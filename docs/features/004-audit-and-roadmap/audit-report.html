02SoundCloud loading — cache + "TUNING…"

Two-pronged: kill the wait with a client cache, and make the unavoidable first wait feel like part of the retro-TV fiction.

The cold waterfall today
click→ OAuth token ~400ms→ /resolve ~500ms→ Zod + filter→ widget load ~400ms→ audio

Playlist cache

localStorage, 12h TTL, versioned key with a purge line in the existing mount-time cleanup. Zod-validated on read (incl. embedUrl format — there were prior cache bugs). Stale-while-revalidate, but staged, never hot-swapped into a playing channel — which also fixes the existing YouTube SWR jump bug.

"TUNING…" overlay

Full-screen analog static (GLSL noise, CSS fallback, static under reduced-motion) with a channel-number OSD and status line driven by one consolidated state machine: RESOLVING SIGNAL… → LOCKING AUDIO… → (play | NO SIGNAL). No new timers.

The decision this forces: music channels become explicitly per-viewer divergent (video channels keep the strict invariant). The cache freezes a per-viewer snapshot, which makes a single viewer's schedule more stable, not less. CLAUDE.md gets updated to state this honestly.

04Roadmap

Three specs, sequenced so each unblocks the next. Security and the CI gate go first; perf clears the path for visuals; visuals ship in three PRs.

  1. Audit Batch A+B — SSRF fix, strip diagnostics, .dockerignore; add CI typecheck + fix @types/youtube, clear tsc errors. security + a gate that stops regressions
  2. Audit Batch C — getAttribLocation cache, 1s-tick memo, perf prep. clears the path for visuals
  3. SoundCloud loading — playlist cache + no-hot-swap SWR + "TUNING…" overlay + consolidated state machine.
  4. Visualizers PR 1 — perf gates, backend interface, single-owner host, lazy states.
  5. Visualizers PR 2 — the GLSL wave: Fractal Voyage, Liquid Ink, Acid Melt, Warp Drive.
  6. Visualizers PR 3 — three.js (Neon Tunnel, Particle Galaxy) + p5 (Flow-Field), lazy-loaded.
  7. Audit Batch D — token-cache tests, SC helper tests, route-file splits, doc fixes. steady-state
